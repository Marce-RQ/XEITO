# Xeito QA Project

Quality Assurance framework for Xeito Sports Management App.

## Project Structure

```
├── test-plans/         # Test charters and exploratory testing guides
├── bug-reports/        # Bug tracking and templates
└── test-data/          # Test accounts and sample data documentation
```

## Current Testing Scope

**Application Under Test**: https://www.xeito.app/  
**Status**: Under active development  
**Ready Sections**: Sign Up/Login, Leagues, Profile

## Testing Approach

Currently focused on **manual exploratory testing**. Automation will be considered once features stabilize.

## Setup & Discovery

### Access & Tools

-   [ ] Register at [Get early access](https://id.xeito.app/knock-knock), and inform the code owner
-   [ ] Access to [Xeito](https://www.xeito.app) app
-   [ ] Request access to repo
-   [ ] Request access to bug tracking tool (Linear)
-   [ ] Join team communication channels (Slack/Discord)
-   [ ] Add to QA team calendar for standups/meetings

## Quick Start
1. Read [QA Onboarding](QA-ONBOARDING.md)
2. Familiarize yourself with the risk assessment and bug report template in `bug-report/`
3. Use exploratory test charters in `test-plans/`
4. Document findings using [Linear](pdfAndImages/Linear.png)
